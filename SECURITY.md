# 🛡 Pharmora Security Policy

## Protecting the open pharmacy learning ecosystem

Security, privacy, and trust are important parts of Pharmora.

---

# Supported Status

Current phase:

🚧 Prototype Development

The current version is a static frontend prototype.

Features not yet implemented:

- User authentication
- Database storage
- Resource uploads
- Personal accounts
- Backend services

Security policies will expand as these features are introduced.

---

# Security Principles

Pharmora development should follow:

- Privacy-first design
- Minimal data collection
- Secure coding practices
- Open review process
- Transparent improvements

---

# Reporting Issues

If you discover:

- Security problems
- Privacy concerns
- Vulnerabilities
- Unsafe behavior

Please report them responsibly through project maintainers.

Avoid publicly exposing details that could harm users before review.

---

# Educational Content Safety

Because Pharmora involves pharmacy education:

Contributors should avoid:

- Incorrect academic information
- Misleading healthcare claims
- Unverified medical advice
- Unsafe recommendations

Educational resources should be:

- Reviewed
- Referenced when possible
- Improved through community feedback

---

# Future Security Goals


## Authentication

Planned protections:

- Secure login
- Strong session management
- Role permissions


## Data Protection

Goals:

- Protect user information
- Limit unnecessary data storage
- Transparent data handling


## Content Moderation

Planned:

- Resource review
- Teacher verification
- Reporting system
- Quality checks


---

# Dependency Security

Future software dependencies should be:

- Open source
- Actively maintained
- Regularly updated

---

# Community Responsibility

Security is a shared responsibility.

Everyone can help by:

- Reporting problems
- Reviewing changes
- Improving documentation
