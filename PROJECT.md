# Pharmora Project Blueprint

> **This document defines what Pharmora is, what it aims to become, and the overall structure of the platform. Unlike `VISION.md`, which explains *why* the project exists, this document explains *what* is being built.*

---

# Project Summary

**Project Name**

Pharmora

**Project Type**

Open-source pharmacy knowledge and collaboration platform.

**Current Stage**

Pharmora Core v3 – Architecture & Foundation.

**Long-term Goal**

Build the world's most comprehensive open platform for pharmacy education, research, professional collaboration, scientific knowledge, and lifelong learning.

---

# Problem Statement

Pharmacy knowledge is fragmented across books, journals, universities, personal notes, websites, organizations, and disconnected communities.

Existing platforms usually solve only one problem:

* Learning
* Research
* Discussion
* File sharing
* Jobs
* Professional networking

Pharmora aims to connect all of these into one ecosystem.

---

# Project Objectives

The project aims to:

* Make pharmacy knowledge easier to discover.
* Preserve valuable educational resources.
* Enable collaboration between the global pharmacy community.
* Build reusable systems instead of isolated features.
* Support students throughout their academic and professional journey.
* Create a sustainable open-source platform.

# Architecture Principles

Pharmora is built around several core principles.

* Entity-first
* Schema-driven
* Relationship-powered
* Static-first
* Modular
* Extensible
* Backward compatible
* Community-driven

---

# Development Principles

Every new feature should:

* Strengthen the platform.
* Reuse existing systems whenever possible.
* Minimize duplication.
* Remain backward compatible.
* Be documented before implementation.
* Be independently testable.

---

# Success Metrics

The project is considered successful when:

* Knowledge becomes easier to discover.
* Community contributions continue to grow.
* Educational resources become more accessible.
* Institutions adopt and contribute to the platform.
* New modules can be built using existing Core systems with minimal custom code.

---

# Non-Goals

Pharmora is **not** intended to become:

* A closed-source platform.
* A proprietary LMS replacement.
* A social media platform.
* A marketplace driven primarily by advertising.
* A platform that locks users into proprietary infrastructure.

---

# Evolution Strategy

Pharmora will evolve in layers.

1. Core Platform
2. Knowledge Platform
3. Community Platform
4. Developer Platform
5. Ecosystem Platform

Each stage builds on the previous one without replacing it.
